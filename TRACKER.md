## V3-13 — Documents → Editor: DOCX formatting fidelity + clean print ☑ (2026-07-06, BE+FE)
**Closes the V3-12 deferred "visual fidelity" gap.** Makes an uploaded DOCX résumé open in the editor close to the original — **ruled section headings, tab-aligned two-column lines, centered contact block, compact spacing** — and fixes **multi-page print**. **Investigation finding that shaped the approach:** the sample DOCX has *no* paragraph borders or styles (verified by parsing `word/document.xml`/`styles.xml`) — the section rules are a résumé *convention*, not encoded data — so the whole backend change is a **pure HTML post-process on mammoth's output**, no `document.xml` walker. **Backend (`engine/extract.js`):** `postProcessDocxHtml()` promotes paragraphs whose normalized text matches a **curated section-label list** (`SUMMARY`, `EXPERIENCE`, `TECHNICAL SKILLS`, … — never a bold job title) to `<h2 data-rule="true">`, and converts **tab-split lines** (the literal `\t` mammoth preserves) into a borderless `<table class="doc-columns">`; `extractDocxHeader` now **centers** contact lines whose source `<w:p>` has `w:jc="center"`. Everything is wrapped so any error falls back to raw mammoth output — **never regresses**; `extractText` (keyword analysis) untouched. **Frontend:** two attribute-preserving TipTap extensions — **`HeadingRule`** (persists `data-rule` on headings) and **`TableColumns`** (re-adds the `doc-columns` class TipTap's Table drops on import) — plus the importer (`htmlToProseMirror`) aligned with the editor schema (Table/TextAlign/HeadingRule/TableColumns, heading levels `[1,2,3]`) and `.tiptap` CSS for the rule, borderless columns, and compact spacing (screen + print). **Print/pagination fix:** each printed page now gets real `@page` margins matching the selected preset (was `margin:0` + sheet padding, which only margined page 1's top and the last page's bottom), the screen padding is zeroed in print, and break-control keeps headings with their content and stops tables/rows/list-items/images splitting across pages. Built **subagent-driven (9 TDD tasks + fixes**, fresh implementer + spec/quality reviewer each) + a final **Opus cross-repo review** (verdict: **Ready to merge**, no XSS surface). **Two defects caught in review/verification and fixed:** a pre-existing `!important` print cell-border was overriding the borderless columns; and the `doc-columns` class was being silently dropped on import — a full BE→FE pipeline probe caught it → the `TableColumns` extension. **BE 217 / 1-skipped, FE 242 tests**; both builds green. **Known limits** (spec): only two-segment tab lines become columns; a non-curated section label imports as a plain bold paragraph; exact font sizes not reproduced; PDF stays plain text. Spec/plan: BE `docs/superpowers/{specs,plans}/2026-07-03-docx-open-fidelity*`.

## V3-10 — Editor v6: Image text-wrapping, drag positioning & free placement ☑ (2026-07-02, merged & pushed to `main`)

(c) **Free-placement relabel** — popup modes renamed/reordered to **In line · Over text · Behind text · Wrap around · Break**, where "Over text" is the clear drag-anywhere mode. Built subagent-driven across the three batches + final **Opus** reviews per batch. Notable review catches (all fixed + re-verified): inline-node NodeSelection collapse after attribute edits (a shared reselect helper restores it); the shallow→recursive migration gap (legacy images in tables/lists were being dropped); a click-not-drag committing a (0,0) reposition; and `repositionImageNode` throwing instead of returning null on an off-target drop. Drag is manual/e2e verified; controller browser-verified the wrap reposition (node moved + side flipped + text wrapped). New pure helpers `imageReposition.js` (`sideForX`, `repositionImageNode`) unit-tested. **221 tests** (full suite green, pristine).
